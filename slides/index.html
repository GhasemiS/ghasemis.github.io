üíª Installing Conda

Downloading and installing **Miniforge** (a minimal Conda installer from the conda-forge community) is simple:


wget https://github.com/conda-forge/miniforge/releases/latest/download/Miniforge3-Linux-x86_64.sh -O miniforge.sh
chmod +x miniforge.sh
bash miniforge.sh -b
./miniforge3/bin/conda init bash

The last step ensures Conda is available when you log in. Open a **new connection** to check if it's available.

‚öôÔ∏è Configuring Conda: Channels

**Conda channels** are similar to repositories for other package managers. We'll add common bioinformatics channels:


conda config --append channels bioconda
conda config --append channels genomedk
conda config --set channel_priority strict

Setting `channel_priority strict` ensures packages from higher-priority channels (like `conda-forge`, which is default) are preferred.

üö´ Configuring Conda: Avoid `base`

  • Conda creates a **base** environment that contains Conda itself.
  • üö® **NEVER** install packages directly into the `base` environment to avoid ruining your installation.

Prevent accidental use of `base` by disabling its automatic activation:


conda config --set auto_activate_base false

üîç Reviewing Conda Configuration

After configuration, your settings are stored in the .condarc file. This is what it should look like:


cat $HOME/.condarc
      

channels:
  - conda-forge
  - bioconda
  - genomedk
channel_priority: strict
auto_activate_base: false

üîé Finding Conda Packages

You can search for packages using the **anaconda.org** website or the command line:


conda search samtools

‚ö†Ô∏è **Be Aware:** The Conda package name may differ from the software's official name (e.g., searching for `biobambam2` might require searching for `biobambam`).

If you can't find a package, contact your administrators for potential creation or alternative methods (like Singularity/Apptainer).

‚ûï Installing Conda Packages

Always create a new environment for your project. Here, we create one for **PySAM**:


conda create -n amazing-project pysam

This creates a clean environment with minimal dependencies. To use the software, you must **activate** the environment:


conda activate amazing-project
(amazing-project) $ python -c 'import pysam; print(pysam.__version__)'

Notice the prompt changes to (amazing-project).

üì¶ Managing Environments & Sharing

Installing More Packages

Install additional packages while the environment is active:


(amazing-project) $ conda install r-ggplot2

Sharing Your Environment

Export your environment specification to allow collaborators to recreate it exactly:


(amazing-project) $ conda env export > environment.yml

üîÑ Recreating an Environment

To recreate an environment from a shared environment.yml file:


conda env create -f environment.yml

This command uses the exact specifications in the file to build an identical environment on a new machine.
